# goit-markup-hw-06-r

Проєкт

«A1» При перегляді сторінки на будь-якому пристрої шириною від 320px, не
з'являється горизонтальна смуга прокручування

«A2» Вихідний код відформатований за допомогою Prettier.

«A3» Всі зображення та текстовий контент взяті з макета.

«A4» Підключений нормалізатор стилів modern-normalize.

«A5» Код написаний з дотриманням настанови.

Респонсивність та адаптивність

«B1» На сторінці в блоці <head> є метатег viewport.

«B2» Верстка виконана щодо трьох точок перелому: 320px, 768px і 1158px.

«B3» Всі фонові і контентні растрові зображення — респонсивні і підтримують
екрани щільністю x1 и x2.

«B4» Для респонсивних фонових зображень використані медіа-функція
min-resolution.

«B5» При написанні стилів використаний Mobile First підхід і медіа-функція
min-width.

«B6» Не потрібно обгортати мобільні стилі в @media (min-width: 320px), оскільки
вони мають бути задані за замовчуванням.

«B7» Стилі, необхідні тільки у певному проміжку, закриті в медіа-запити
(min-width: ) and (max-width: ) або тільки (max-width: ).

«B8» В медіа-запитах відсутнє зайве дублювання стилів.

«B9» Для реалізації адаптивної графіки використовувати src + srcset (примітка:
не picture + source + img).

Мобільне меню

«C1» Мобільне меню має бути реалізоване як окремий елемент (вікно/оверлей),
розміщений у кінці body, після футера.

«C2» Виконано оформлення всіх елементів мобільного меню.

«C3» Ширина і висота меню займає весь в'юпорт.

«C4» Спочатку меню приховано.

«C5» Якщо додати на елемент меню клас is-open, воно з'являється.

«C6» Навігація розташовується зверху, а контакти та соцмережі — внизу меню,
розділені гнучким відступом, що адаптується до висоти екрана.

«C7» Меню відображається лише в мобільній версії; на планшетах і десктопах воно
має бути приховане.

«C8» Сині акцентні кольори в посиланні навігації та контактів мають бути
статичними та відображатися постійно, без зміни стану.

Стилізація

«D1» Оверлей на картках портфоліо має бути на всіх ширинах.

«D2» Інтерактивні елементи повинні мати стилі :hover та :focus незалежно від
розміру екрана.

«D3» У секції hero для кожного брейкпоінта (mobile / tablet / desktop) має
використовуватися окреме фонове зображення з відповідними обмеженнями ширини:
max-width 320px / 768px / 1440px.

«D4» У секціях Our Team та Footer використовується нестандартна ширина
контейнера: 584px (552px контент + по 16px паддінги) або 768px (552px контент +
по 108px паддінги).

«D5» Контактна інформація в хедері має різні типографічні стилі на tablet і
desktop брейкпоінтах (зміна font-size, line-height тощо).

«D6» У секції Feature вирівнювання та типографічні стилі заголовків і параграфів
відрізняються залежно від розміру екрана.
